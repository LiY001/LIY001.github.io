第二章 关系数据库

定义

域:一组具有相同数据类型的值的集合
笛卡儿积:域上的一种集合运算

关系代数

并:R∪S
差:R-S
交:R∩S
笛卡儿积:R×S
选择
投影
连接:也称为θ连接
自然连接:是一种特殊的等值连接,用 R ⋈ S 来表示
        先等值连接,再去掉重复的列
当两个关系没有公共属性时,则自然连接为笛卡儿积

除 : R÷S 。R中关系包含S中的所有关系,则列出R中除去S表中其余的几列

外连接

左外连接（d）:取出左侧关系中所有与右侧关系的任一元组都不匹配的元组,用空值填充所有来自右侧关系的属性
右外连接（e）
全外连接（f）:左外连接和右外连接和并

3.3 查询

SELECT 目标列表达式     ---需要那些列
FROM 表名或视图名       ---来自那些表
[WHERE<条件表达式>];    ---条件
<目标列表达式>中各个列的先后顺序可以和表中的顺序不一致
查询全部列:在目标列表达式中写星号“ * ”
<目标列表达式>可以是表中存在的属性列,也可以是表达式、字符串常量或函数
函数 datediff(datepart,date1,date2)   返回date1和date2的时间间隔,其单位由datepart决定

例3.11 查询所有抢修工程的抢修天数

SELECT prj_name, start_date, end_date, datediff(day,start_date,end_date)
FROM Salvaging;

去掉表中的重复行

SELECT DISTINCT 名称
FROM 名称;

确定范围的查询

BETWEEN...AND
NOT BETWEEN...AND

例3.16 查询单价在50~100的物资名称、数量及其单价

SELECT mat_name,amount,unit
FROM Stock
WHERE unit BETWEEN 50 AND 100;

字符匹配的查询

[NOT] LINK '匹配符'
--- % (百分号)代表任意长度的字符串。例 a%b:表示以a开头,以b结尾的任意长度的字符串
--- _ (下横线)代表任意单个字符。例 a_b:表示以a开头,以b结尾的长度为3的任意字符串

3. 对查询结果进行排序

ORDER BY 列名

4. TOP 子句的用法

放在 SELECT 后
TOP n 子句:在查询结果中输出前面的n条记录
TOP n PERCENT 子句:在查询结果中输出前面占结果记录总数的n% 条记录

5. 聚集函数

放在 SELECT 后
COUNT (DISTINCT|ALL * )     统计表中元组个数
COUNT 列名                  统计一列中值的个数
SUM 列名                    求和
AVG 列名                    求平均值
MAX 列名                    求最大值
MIN 列名                    求最小值
DISTINCT 短语表示在计算时要取消指定列中的重复值

3.3.2 连接查询

等值与非等值连接查询

表名1.列名1 比较运算符 表名2.列名2;
或 表名1.列名1 BETWEEN 表名2.列名2 AND 表名2.列名3;
比较运算符 ＝ 、＞ 、＜ 、＞＝ 、＜＝ 、！＝(或＜＞)
＝ 是 等值连接,其他为非等值连接
在等值连接中把目标列中重复的属性去掉,则为自然连接

外连接查询

左外连接:LEFT OUTER JOIN
右外连接:RIGHT OUTER JOIN
全外连接:FULL OUTER JOIN

复合条件连接查询

WHERE 子句中可以有多个连接条件,称为复合条件查询

例3.43 查询项目号为“20100015”的抢修项目所使用的物资编号、物资名称、规格和使用数量

SELECT Out_stock.mat_num,mat_name,speci,Out_stock.amount
FROM Stock,Out_stock
WHERE Stock.mat_num = Out_stock.mat_num AND prj_num = '20100015'

自身连接查询

同一个表与自己进行连接
必须给表取别名加以区分,必须加前缀

例3.45 查询同时使用了物资编号为m001和m002的抢修工程的工程号

SELECT A.prj_num
FROM Out_stock A, Out_stock B
WHERE A.prj_num = B.prj_num AND A.mat_num = 'm001' AND B.mat_num = 'm002';

3.3.3 嵌套查询

子查询的 SELECT 语句不能使用 ORDER BY 子句,ORDER BY 子句只能对最终查询结果排序

带谓词 IN 的嵌套查询

例3.47 查询工程项目“观澜站光缆抢修”所使用的物资编号和物资名称

SELECT mat_num,mat_name
FROM Stock
WHERE mat_num IN
      (SELECT mat_num
       FROM Out_stock
       WHERE prj_num IN
          (SELECT prj_num
           FROM Salvaging
           WHERE prj_name = '观澜站光缆抢修'));

带有 ANY 或 ALL 谓词的子查询

> ANY  大于子查询结果中的某个值
> ALL  大于子查询结果中的所有值...

例3.49 查询其他仓库中比供电局 1# 仓库的某一物资库存量少的物资名称、规格和数量

SELECT mat_name,speci,amount
FROM Stock
WHERE warehouse <> '供电局 1# 仓库'
        AND amount < ANY
          (SELECT amount
           FROM Stock
           WHERE warehouse = '供电局 1# 仓库');

3.3.4. 集合查询

并操作 UNION
交操作 INTERSECT
差操作 EXCEPT
集合操作会自动去除重复元组,保留时要用 ALL 关键词

查询存放在供电局 1# 仓库的物资及单价不大于50的物资

SELECT *
FROM Stock
WHERE warehouse = '供电局 1# 仓库'
UNION
SELECT *
FROM Stock
WHERE unit <= 50;

3.5 视图

定义视图

CREATE VIEW 视图名 [列名,列名...]
AS 子查询
[WITN CHECK OPTION] ;
---子查询可以是任意的 SELECT 语句,但不允许含有 ORDER BY 子句
---组成视图的属性列名要么全部省略,要么全部指定

例3.71 建立供电局 1# 仓库所存放物资的视图

CREATE VIEW s1_stock
AS
 SELECT mat_num,mat_name,speci,amount,unit
 FROM Stock
 WHERE warehouse = '供电局 1# 仓库';

第五章

定义

包含在任何一个候选码中的属性称为 主属性
不包含在任何码中的属性称为 非主属性或非码属性
主属性是表格的第一列
规范化:消除关系模式中的数据冗余

第一范式（1NF）

一个关系模式R的所有属性均为简单属性,即每个属性都是不可再分的,记作R∈1NF

第二范式（2NF）

满足第一范式,且所有非主属性都完全依赖于主码
即:消除非主属性对主属性的部份依赖

第三范式（3NF）

满足第二范式,且所有非主属性对任何候选码都不存在传递依赖
即:消除非主属性对主属性的部份依赖和传递依赖

第六章 索引

聚集索引

数据文件中的记录按照索引键指定的顺序排序
一个数据表只能建立一个聚集索引

非聚集索引

一个表可以有多个非聚集索引

唯一索引

不允许表中任何两行具有相同索引值的索引

创建索引

CREATE 索引类型
INDEX 索引名称 ON 用于创建索引的表或视图 (用于创建索引的列 指定索引列的排列方式,ASC升序、DESC降序)
--- UNIQUE :创建一个唯一索引
--- CLUSTERED :聚集索引
--- NONCLUSTERED :非聚集索引

查看索引

USE 数据库名
Exec  sp_helpindex 所查表

删除索引

DROP INDEX 索引名

第七章 E-R图

实体-联系模型(E-R模型)

实体可以是物理存在的事物,也可以是抽象的概念
每个实体都有一组特征或性质,称为实体的属性
由一个属性构成的键称为简单键
由多个属性构成的键称为复合键

实体间的联系

一对一联系(1:1)
一对多联系(1:n)
多对多联系(m:n)

实体-联系图

(1).实体集:用矩形表示,矩形框内写明实体名
(2).属性:用椭圆表示,并用无向边将其与相应的实体集连接起来;
     多值属性用双层的椭圆表示
     主键加下划线
(3).联系:用菱形表示,菱形框内写明联系的名称,并用无向边分别与有关实体集连接起来;同时在无向边上表明联系的类型(1:1、1:n、m:n)
